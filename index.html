📋 Antes de empezar

Requisitos previos

Asegúrate de tener todo esto listo antes de seguir la guía.

Home Assistant funcionando en tu red local

Accesible por HTTP (ej: http://192.168.1.100:8123). No necesitas HTTPS ni certificados SSL.

Google Nest Hub 2 en la misma red Wi-Fi

Tu Nest Hub debe estar configurado con la app Google Home y conectado a la misma red local que tu servidor de HA.

HACS instalado en Home Assistant

Home Assistant Community Store. Necesario para los métodos 1 y 2. Si no lo tienes, instálalo desde hacs.xyz.

IP fija para tu Nest Hub

Entra en la configuración de tu router y asigna una IP estática a tu Nest Hub. La encontrarás en la app Google Home → Dispositivo → Información del dispositivo.

🏆 Método recomendado

01 Continuously Casting Dashboards

Integración HACS que mantiene tu dashboard casteado automáticamente. Se auto-recupera si el Nest Hub entra en reposo o si alguien interrumpe el cast.

Crear un usuario dedicado para el Nest Hub

Ve a Ajustes → Personas → Añadir persona y crea un usuario nuevo. Importante: sin permisos de administrador. Anota el User ID que verás en la URL al editar el perfil.

💡 El User ID es la cadena larga que aparece en la URL cuando estás en /profile/. Ejemplo: a1b2c3d4e5f6...

Configurar Trusted Networks en configuration.yaml

Esto permite que el Nest Hub inicie sesión sin contraseña desde su IP fija.

homeassistant:
  auth_providers:
    - type: trusted_networks
      trusted_networks:
        - 192.168.1.XX/32  # ← IP fija de tu Nest Hub
      trusted_users:
        192.168.1.XX:
          - "TU_USER_ID_AQUI"
      allow_bypass_login: true
    - type: homeassistant  # ← mantener login normal para otros
⚠️ Reinicia Home Assistant después de modificar configuration.yaml. Si algo va mal, borra los cambios y reinicia de nuevo.

Instalar ha-catt-fix (evita desconexión a los 10 min)

Este recurso frontend previene que el Nest Hub desconecte el cast por inactividad.

1. HACS → Frontend (pestaña)
2. Tres puntos (⋮) → Custom repositories
3. URL: swiergot/ha-catt-fix
   Categoría: Dashboard (Lovelace)
4. Buscar "ha-catt-fix" → Descargar
5. Verificar en: Dashboard → Editar → Recursos

Instalar Continuously Casting Dashboards

1. HACS → Integraciones
2. Tres puntos (⋮) → Custom repositories
3. URL: b0mbays/continuously_casting_dashboards
   Categoría: Integration
4. Buscar → Descargar → Reiniciar HA

Crear un dashboard optimizado para el Nest Hub

La pantalla del Nest Hub 2 es de solo 7", así que necesitas un dashboard limpio y táctil.

  • Ve a Ajustes → Dashboards → Añadir dashboard
  • Dale un nombre como nest-hub
  • Usa Grid Cards y tarjetas grandes, con botones táctiles fáciles de pulsar
  • Instala kiosk-mode vía HACS para ocultar la barra superior (añade ?kiosk a la URL)

Configurar la integración

Añade esto a tu configuration.yaml:

continuously_casting_dashboards:
  logging_level: warning
  cast_delay: 45          # segundos entre comprobaciones
  start_time: "07:00"     # hora de inicio del cast
  end_time: "01:00"       # hora de fin del cast
  devices:
    "Pantalla del salón":  # ← nombre EXACTO en Google Home
      - dashboard_url: "http://192.168.1.100:8123/lovelace-nest/0?kiosk"
        volume: 5
ℹ️ Usa la IP local de tu HA en la URL del dashboard, no homeassistant.local. El nombre del dispositivo debe coincidir exactamente con el que aparece en la app Google Home.

Reiniciar y verificar

Reinicia Home Assistant. La integración debería empezar a castear automáticamente dentro de la ventana horaria configurada. Puedes controlarla con servicios:

# Parar el cast de un dispositivo
service: continuously_casting_dashboards.stop_casting
data:
  device_name: "Pantalla del salón"

# Reanudar
service: continuously_casting_dashboards.resume_casting
data:
  device_name: "Pantalla del salón"
🔁 Alternativa

02 DashCast

Componente custom de AlexxIT que permite castear cualquier URL a tus Chromecasts mediante un servicio de HA. Más ligero, pero necesita automatizaciones para mantenerlo vivo.

Configurar Trusted Networks

Igual que en el Método 1, paso 2. Configura la IP de tu Nest Hub como red de confianza.

Instalar DashCast vía HACS

1. HACS → Integraciones → ⋮ → Custom repositories
2. URL: AlexxIT/DashCast
   Categoría: Integration
3. Descargar → Reiniciar HA
4. Ajustes → Integraciones → Añadir integración → DashCast

Crear automatización para castear periódicamente

Como el Nest Hub entra en reposo tras ~10 minutos, necesitamos re-castear cada 9 minutos:

automation:
  - alias: "Cast Dashboard al Nest Hub"
    trigger:
      - platform: time_pattern
        minutes: "/9"
    condition:
      - condition: time
        after: "07:00:00"
        before: "01:00:00"
    action:
      - service: dash_cast.load_url
        data:
          entity_id: media_player.nest_hub_salon
          url: "http://192.168.1.100:8123/lovelace-nest/0"

03 CATT (Cast All The Things)

Herramienta CLI en Python que castea URLs a dispositivos Chromecast. Necesitas una máquina Linux/Docker en tu red donde ejecutarlo.

Instalar CATT

pip install catt

Escanear dispositivos

catt scan
# Salida: Pantalla del salón - 192.168.1.55 - Google Nest Hub

Probar el cast manualmente

catt -d "Pantalla del salón" cast_site \
  http://192.168.1.100:8123/lovelace-nest/0

Automatizar con shell_command en HA

# configuration.yaml
shell_command:
  cast_nest_hub: 'catt -d "Pantalla del salón" cast_site http://192.168.1.100:8123/lovelace-nest/0'

Luego crea una automatización que llame a shell_command.cast_nest_hub cada 9 minutos, igual que en el método DashCast.

Script anti-reposo (alternativa)

Si prefieres un script independiente fuera de HA:

#!/bin/bash
while true; do
  if ! catt -d "Pantalla del salón" status | grep 'PLAYING'; then
    catt -d "Pantalla del salón" volume 5
    catt -d "Pantalla del salón" cast_site \
      http://192.168.1.100:8123/lovelace-nest/0
  fi
  sleep 540  # 9 minutos
done

Tips y optimizaciones

Instala kiosk-mode para un look limpio

El complemento kiosk-mode de HACS oculta la barra superior, sidebar y otros elementos de la UI. Añade ?kiosk al final de la URL del dashboard para activarlo.

Diseña dashboards específicos para 7 pulgadas

  • Usa Grid Cards para organizar el layout
  • Botones grandes (mínimo 48px de alto) para uso táctil
  • Fuentes legibles, evita textos pequeños
  • Muestra solo lo esencial: reloj, clima, luces, escenas

Baja el volumen del "bip" de cast

Cada vez que se inicia un cast, el Nest Hub hace un sonido. Puedes bajar el volumen antes de castear usando el parámetro volume en CCD o con:

service: media_player.volume_set
data:
  entity_id: media_player.nest_hub_salon
  volume_level: 0.05

Usa temas oscuros para no deslumbrar de noche

Configura un tema oscuro para el usuario del Nest Hub o usa una automatización que cambie el tema según la hora del día.

Solución de problemas comunes

  • Pantalla en blanco: Verifica que Trusted Networks está bien configurado y reinicia HA
  • Se desconecta a los 10 min: Instala ha-catt-fix (Método 1, paso 3)
  • No encuentra el dispositivo: El nombre debe ser EXACTO al de Google Home, incluidas mayúsculas
  • Login loop: Asegúrate de que allow_bypass_login: true está configurado
